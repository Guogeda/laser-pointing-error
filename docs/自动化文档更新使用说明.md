# 自动化文档更新使用说明

## 概述

这个方案实现了在多轮对话后自动总结并更新项目文档的功能。它由两部分组成：

1. **Hookify 提醒规则**：在对话结束时提醒你更新文档
2. **Python 自动化脚本**：实际执行文档更新操作

---

## 文件说明

| 文件 | 位置 | 说明 |
|------|------|------|
| Hookify 规则 | `.claude/hookify.auto-document-update.local.md` | 对话结束时自动提醒 |
| 更新脚本 | `src/update_docs.py` | 实际更新文档的 Python 脚本 |
| 使用说明 | `docs/自动化文档更新使用说明.md` | 本文档 |

---

## 快速开始

### 1. 正常对话结束后使用

当你完成一轮对话，想要更新文档时，直接运行：

```bash
python src/update_docs.py
```

脚本会引导你通过交互式输入本次对话的内容。

### 2. Hookify 自动提醒

每次对话结束时，Hookify 会自动显示：

```
📝 **对话结束提醒**

在结束之前，请考虑运行文档更新脚本：

```bash
python src/update_docs.py
```

这个脚本会：
1. 自动总结本次对话内容
2. 更新 @需求文档-new.md
3. 更新 @CLAUDE.md
4. 更新 @findings.md
5. 更新 @progress.md
6. 更新 @task_plan.md
```

---

## 交互式输入说明

运行脚本时，它会询问以下信息：

| 输入项 | 说明 | 示例 |
|--------|------|------|
| 🔍 新发现 (Research Findings) | 本次对话中的重要发现 | 发现了32star数据的新特征 |
| 📋 完成的动作 (Actions) | 你做了什么工作，多个用分号分隔 | 创建了卫星分组配置文件;重构了verify_complete.py |
| 📁 修改的文件 (Files) | 这次改动了哪些文件，多个用分号分隔 | src/config/satellite_groups.py;src/verify_complete.py |
| ✅ 完成的任务 (Tasks) | task_plan.md中完成的任务，多个用分号分隔 | 创建卫星分组配置文件;创建jg02终端配置 |
| 📌 CLAUDE.md 备注 | 需要在CLAUDE.md中记录的内容 | 添加了jg02组卫星支持说明 |
| 📋 需求文档更新 | 需要在需求文档中记录的变更 | v1.2更新 - 删除链路配对分析章节 |

## 支持更新的文档

### 1. findings.md
- 在 "Research Findings" 部分添加新的发现记录

### 2. progress.md
- 在当前 Phase 的 "Actions taken" 部分记录完成的动作
- 在 "Files created/modified" 部分记录修改的文件

### 3. task_plan.md
- 将待办任务标记为已完成（从 "[ ]" 改为 "[x]"）

### 4. CLAUDE.md
- 在 "注意事项" 部分或文件末尾添加备注

### 5. 需求文档-new.md
- 在 "版本历史" 部分添加新的版本记录

## 配置 Hookify 规则

### 查看已配置的规则
```bash
/hookify:list
```

### 编辑规则
直接编辑文件：`.claude/hookify.auto-document-update.local.md`

### 启用/禁用规则
在文件的 frontmatter 中设置：
```yaml
enabled: true    # 启用
enabled: false   # 禁用
```

### 删除规则
直接删除文件：`.claude/hookify.auto-document-update.local.md`

## 注意事项

1. **Hookify 规则位置**：规则文件必须放在 `.claude/` 目录下
2. **实时生效**：修改 Hookify 规则后立即生效，无需重启
3. **备份**：建议在运行脚本前备份重要文档
4. **交互式模式**：在非交互环境中使用时可能会出错，建议在终端中运行

---

## 故障排除

### 问题：脚本运行时出现编码错误

**解决方法**：确保你的终端支持 UTF-8 编码

### 问题：Hookify 没有提醒

**检查点**：
1. 确认规则文件在 `.claude/` 目录下
2. 确认 `enabled: true`
3. 确认 `event: stop`

### 问题：找不到文件路径

**解决方法**：从项目根目录运行脚本
```bash
cd "c:/Users/17251/Desktop/激光指向误差计算"
python src/update_docs.py
```
